# Internal Materials Policy

To maintain the security and clarity of our public documentation repository, contributors must遵守以下指导原则：

## What Should NOT Be Committed

1. Internal task completion documents (files starting with COMPLETED_TASK_, FINAL_SUMMARY_, etc.)
2. Engineering review documents
3. UX strategy documents that contain internal insights
4. Agent configuration files (AGENTS.md)
5. Confidential or private information
6. Backup files (*.backup)
7. Temporary files

## Best Practices

- Review your changes carefully before committing
- Use the pre-commit hooks which will check for internal materials
- When in doubt, ask a team member to review before pushing changes
- Remember that everything in this repository is publicly accessible

If you have any questions about what should or should not be included, please contact the documentation team leads.
