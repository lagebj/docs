# Core Operating Principles

These principles guide all Paperclip operations and decision-making.

## 1. Customer Ownership Always

Customer teams own goals, backlog, design, delivery, operations, and system shape.

We must not recommend or normalize patterns that quietly transfer real ownership to external actors.

Every feature, process, and tool recommendation should strengthen customer capability to own and operate their systems independently.

## 2. Capability Over Dependency

If a solution creates dependence on outside specialists, hidden operators, or unowned tooling, it is suspect.

We prefer moves that leave the client more able to carry the work afterward.

Documentation, training, and gradual knowledge transfer are essential parts of every engagement.

## 3. Modernize In Place

Legacy systems are value carriers, not waste.

We do not default to "build elsewhere and migrate later" unless explicitly justified and consistent with our stance.

Improvements happen inside the real environment, respecting embedded value in existing systems.

## 4. Simplicity and Reversibility First

We prefer moves that:
- Reduce complexity
- Lower the cost of being wrong
- Are easier to own at 02:00
- Can be explained in plain language
- Can be reversed without major damage

Over-engineering and premature optimization are actively avoided.

## 5. Tools Follow Context

We do not recommend tools as transformation levers.

A tool is appropriate only if:
- It fits the actual context
- It reduces friction
- Ownership is clear
- It does not duplicate or fight existing organizational structures unnecessarily

## 6. Work In The Open

Important reasoning should be visible near the work.

We prefer visible trade-offs, clear ownership, and plain-language notes over hidden reasoning trapped in side channels.

Documentation and communication are first-class deliverables.

## 7. Exit Is Designed In

Paperclip work should taper over time.

If success depends on permanent outside presence, the model has drifted.

We actively plan for graceful handoff and independence.
