# Change Management Framework

We approach organizational change by respecting existing structures while introducing improvements that increase capability and reduce dependency on external actors.

## Change Principles

### Respect for Current State


- Acknowledge existing systems contain valuable knowledge and investment
- Understand organizational dynamics before proposing changes
- Work within current incentive structures rather than ignoring them

### Gradual Improvement


- Prefer incremental enhancements over dramatic transformations
- Test changes in small scopes before broader implementation
- Validate learning at each stage before proceeding

### Capability Building


- Every change should increase customer self-sufficiency
- Transfer knowledge and skills throughout implementation
- Develop internal champions for ongoing improvement

## Implementation Approach

1. **Context Analysis**: Understand current workflows, pain points, and constraints
2. **Opportunity Identification**: Find high-leverage improvement points
3. **Solution Design**: Create approaches that align with organizational capacity
4. **Pilot Testing**: Validate changes in limited scope with real users
5. **Gradual Expansion**: Scale successful changes methodically
6. **Capability Transfer**: Ensure customer teams can sustain improvements

## Decision Points

- Choose changes that fit organizational culture and capacity
- Prefer approaches that strengthen rather than bypass existing structures
- Invest in training and documentation proportional to change impact
- Plan explicit transition points from Paperclip-led to customer-led activities
- Establish feedback mechanisms for continuous improvement

## Anti-Patterns to Avoid

- Imposing changes without organizational readiness
- Ignoring political realities in favor of technical optimality
- Creating dependencies rather than building self-sufficiency
- Moving too fast for sustainable adoption
- Failing to validate that changes stick beyond initial implementation
